What’s my preferred image coordinate system?

Cong Wang

10:36:44 PM Sep 22

The right-handed Cartesian coordinate system has been vastly employed in defining laws of physics and math. The horizontal axis is the x-axis, whereas the vertical axis is the y-axis. It is intuitive to apply this coordinate system to an image, as shown in Figure 1. However, this is not the typical image coordinate system.

There are two flavors for an image coordinate system. Both flavors prefer to set the image origin at the top-left corner. The difference is the choice of handedness. The right-handed flavor is equivalent to a \(90^\circ\) clockwise rotation of the traditional right-handed Cartesian coordinate system, as depicted in the middle panel of Figure 2. The left-handed flavor is formed by exchanging two axes, as described in the rightmost panel of Figure 2. My personal choice is the right-handed flavor. Because I always consider the first axis to be the x-axis, which is consistent with the matrix indexing rule adopted in Python (Python is my go-to). However, the computer vision package – opencv endorses the left-handed flavor, which appoints the horizontal axis to be its x-axis.

Caveat to use opencv

If I want to use opencv, I have to convert the image coordinates from the right-handed to the left-handed using the following naive function.

def crd_img2cv(crd):
    ''' Convert coordinate from image format to opencv format.
    '''
    return crd[1], crd[0]
